// SCENARIO

架空のECサイト「ShopDB」のデータを使って、SQLの基礎を練習します。
以下のテーブル構造を確認して、各問題に答えてください。
※ 実際のDB環境がなくても、SQLを書いて答え合わせができます。

// テーブル定義
customers
  • customer_idINT PK
  • nameVARCHAR
  • emailVARCHAR
  • cityVARCHAR
  • ageINT
products
  • product_idINT PK
  • nameVARCHAR
  • categoryVARCHAR
  • priceDECIMAL
  • stockINT
orders
  • order_idINT PK
  • customer_idINT FK
  • product_idINT FK
  • quantityINT
  • order_dateDATE
  • statusVARCHAR
1
全顧客を取得する
customersテーブルの全ての列・全ての行を取得してください。
SELECT
▶
// 期待される出力（例）
customer_idnameemailcityage
1田中太郎tanaka@...東京32
2鈴木花子suzuki@...大阪25
... 全行
// あなたの答え
* はすべての列を表します。テーブル名の前に FROM を使います。
SELECT * FROM customers;
2
商品名と価格だけ取得する
productsテーブルから、nameとpriceの列だけを取得してください。
SELECT
▶
// 期待される出力（例）
nameprice
ワイヤレスイヤホン8900
スマートウォッチ32000
...
// あなたの答え
SELECT のあとに取得したい列名をカンマで区切って書きます。
SELECT name, price FROM products;
3
東京の顧客だけ取得する
customersテーブルから、city が '東京' の顧客を全列取得してください。
SELECT WHERE
▶
// 期待される出力（例）
customer_idnamecity
1田中太郎東京
5山田二郎東京
// あなたの答え
WHERE 列名 = '値' で条件を指定します。文字列はシングルクォートで囲みます。
SELECT * FROM customers WHERE city = '東京';
4
価格が10,000円以上の商品を取得する
productsテーブルから price が 10000 以上の商品を全列取得してください。
SELECT WHERE
▶
// 期待される出力（例）
product_idnameprice
2スマートウォッチ32000
4タブレット58000
// あなたの答え
数値の比較には >= （以上）を使います。
SELECT * FROM products WHERE price >= 10000;
5
ステータスが「shipped」の注文を取得する
ordersテーブルから status が 'shipped' の注文を全列取得してください。
SELECT WHERE
▶
// 期待される出力（例）
order_idcustomer_idstatus
32shipped
75shipped
// あなたの答え
WHERE status = 'shipped' のように文字列条件を指定します。
SELECT * FROM orders WHERE status = 'shipped';
6
顧客名と注文IDを一緒に取得する（JOIN）
customers と orders を結合して、顧客の name と order_id を取得してください。
SELECT JOIN
▶
// 期待される出力（例）
nameorder_id
田中太郎1
鈴木花子3
...
// あなたの答え
JOIN テーブル名 ON 共通キー で結合します。customers.customer_id = orders.customer_id が共通キーです。
SELECT customers.name, orders.order_id FROM customers JOIN orders ON customers.customer_id = orders.customer_id;
7
顧客名・商品名・注文数を取得する（3テーブルJOIN）
customers・orders・products を結合して、顧客名(name)・商品名(name)・数量(quantity)を取得してください。
SELECT JOIN
▶
// 期待される出力（例）
顧客名商品名quantity
田中太郎ワイヤレスイヤホン2
鈴木花子スマートウォッチ1
// あなたの答え
同名の列(name)は テーブル名.name で区別します。JOIN を2回使います。orders → customers と orders → products をそれぞれ結合します。
SELECT customers.name, products.name, orders.quantity FROM orders JOIN customers ON orders.customer_id = customers.customer_id JOIN products ON orders.product_id = products.product_id;
8
大阪の顧客の注文を取得する（JOIN + WHERE）
customers と orders を結合して、city が '大阪' の顧客の order_id と order_date を取得してください。
SELECT JOIN WHERE
▶
// 期待される出力（例）
order_idorder_date
32024-03-15
92024-04-02
// あなたの答え
JOIN の後に WHERE を追加して、city = '大阪' で絞り込みます。
SELECT orders.order_id, orders.order_date FROM orders JOIN customers ON orders.customer_id = customers.customer_id WHERE customers.city = '大阪';
